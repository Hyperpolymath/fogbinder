## Palimpsest License (Philosophical Overlay)

**Version 1.0**

### Preamble

A palimpsest is a manuscript page that has been written on, scraped off, and used again. The earlier writing remains visible beneath the new text, creating layers of meaning.

Fogbinder embraces this philosophy: code is not a single truth, but **layers of interpretation, contradiction, and evolving understanding**.

### Philosophical Commitments

By using Fogbinder, you acknowledge and embrace:

#### 1. **Epistemic Humility**
- Knowledge is provisional, not absolute
- Uncertainty is valuable, not a flaw
- Contradictions invite deeper analysis

#### 2. **Ordinary Language**
- Meaning emerges from use, not definition
- Language games shape understanding
- Context matters more than formal logic

#### 3. **Code as Palimpsest**
- Code has layers of meaning
- Comments, commits, and history are part of the text
- Previous versions inform current understanding

#### 4. **Embrace Contradiction**
- Contradictions are not always errors
- Different frameworks can be incommensurable
- Multiple truths can coexist

#### 5. **Open Inquiry**
- Question assumptions
- Follow the argument where it leads
- Prioritize understanding over winning

#### 6. **Accessibility**
- Software should be usable by all
- Screen readers, keyboard navigation, high contrast
- No user left in the fog

#### 7. **Privacy & Autonomy**
- No surveillance, no tracking, no data collection
- Users control their own data
- Offline-first by design

#### 8. **Community Over Profit**
- Software freedom > commercial gain
- Share knowledge freely
- Contribute upstream

### Legal Effect

The Palimpsest License is a **philosophical overlay**. It has **no legal force** beyond the chosen legal license (MIT or AGPL).

It is a **social contract**, not a legal contract.

It expresses the **spirit** of the project, not the **letter** of the law.

### Violations

If someone violates the Palimpsest principles:
- They do not violate the legal license
- They may lose community goodwill
- The project may fork, disassociate, or criticize

**But:** We will not sue, threaten, or harass. We will engage in **rational discourse**.

## FAQ

### Can I use Fogbinder in commercial software?
- **MIT**: Yes, with attribution
- **AGPL**: Yes, but must provide source code to users

### Can I fork Fogbinder and keep it proprietary?
- **MIT**: Yes
- **AGPL**: No (must remain AGPL)

### What if I only use Fogbinder as a library?
- **MIT**: No obligations beyond attribution
- **AGPL**: Must provide source code if distributed

### What if I run Fogbinder as a web service?
- **MIT**: No obligations
- **AGPL**: Must provide source code to users (network copyleft)

### Do I have to follow the Palimpsest principles?
- **Legally**: No
- **Philosophically**: We hope you will

### Can I change the license of my fork?
- **MIT-licensed fork**: Yes, to any license (MIT, AGPL, proprietary, etc.)
- **AGPL-licensed fork**: No, must remain AGPL

### What if I contribute code to Fogbinder?
- Your contribution is dual-licensed MIT OR AGPL
- You retain copyright
- You grant the project permission to use your code under both licenses
